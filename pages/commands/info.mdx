# Information Commands

Get helpful information about ZeroDegree, your server, and various bot features using these informational commands.

## /help

Display comprehensive help information and command listings.

### Usage
```
/help
```

### Features

**Interactive Help System:**
- Command categories with dropdown menu
- Detailed command descriptions
- Usage examples and syntax
- Permission requirements
- Related command suggestions

**Help Categories:**
- **Configuration** - Server setup commands
- **Moderation** - User management tools
- **Anti-Nuke** - Security and protection
- **Auto-Moderation** - Automated filtering
- **Utility** - Helpful tools and features
- **Information** - Bot and server info

### Help Menu Navigation

**Category Selection:**
Use the dropdown menu to browse different command categories:

1. Select a category from the dropdown
2. View category overview and command list
3. Get detailed information for each command
4. See usage examples and requirements
5. Access related command suggestions

**Command Information:**
Each command entry includes:
- Command syntax and options
- Required permissions
- Usage examples
- Related features
- Configuration tips

### Examples

```
/help
```

The help command will display:
- Welcome message with bot overview
- Interactive category selector
- Quick navigation options
- Support server link
- Documentation resources

### Permissions Required
- None - Available to all users

## /ping

Check the bot's response time and connection status.

### Usage
```
/ping
```

### Information Displayed

**Response Metrics:**
- **Bot Latency** - Time to process and respond to commands
- **API Latency** - Discord API response time
- **WebSocket Ping** - Real-time connection quality
- **Database Ping** - Database connection speed

**Status Indicators:**
- 🟢 **Excellent** (0-100ms) - Optimal performance
- 🟡 **Good** (100-300ms) - Normal performance  
- 🟠 **Slow** (300-500ms) - Degraded performance
- 🔴 **Poor** (500ms+) - Connection issues

**Additional Information:**
- Bot uptime
- Last restart time
- Current server count
- Memory usage

### Use Cases

**Performance Monitoring:**
- Check if bot is responsive
- Diagnose connection issues
- Monitor service quality
- Troubleshoot command delays

**Server Health:**
- Verify bot connectivity
- Check database status
- Monitor API responsiveness
- Track uptime statistics

### Examples

```
/ping
```

**Example Response:**
```
🏓 Pong!

Bot Latency: 45ms 🟢
API Latency: 67ms 🟢
WebSocket: 23ms 🟢
Database: 12ms 🟢

Uptime: 2d 14h 32m
Status: All systems operational
```

### Permissions Required
- None - Available to all users
